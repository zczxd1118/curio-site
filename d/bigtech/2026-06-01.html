3. AlphaEvolve：Gemini 驱动的代码进化 Agent 在多领域达成可验证突破

[大厂 AI 动态] · ⭐⭐⭐⭐⭐ · Hacker News / DeepMind Blog

DeepMind 公布 AlphaEvolve 的扩展报告——一个让 Gemini 写代码并通过自动评估器迭代优化的 coding agent，已经在数据中心调度、矩阵乘法、芯片设计等多个领域产出了可投产的算法。它不是"AI 帮你写函数"，是"AI 跑大量候选实现并用真实评测器筛出更优解"，本质上是把强化学习 + LLM + 代码合成串成一个可商用流水线。

✅ 已确认 ⚖️ 尚属判断
AlphaEvolve 基于 Gemini 模型 + 进化式搜索 + 自动评估器架构 外部团队是否能在没有 Google 评估基础设施的情况下复现存疑
已在 Google 数据中心调度上节省 ~0.7% 全局算力（已部署） 未开源，目前仅以"Impact Report"形式给数据，无 paper 复审
在 Strassen 矩阵乘法的 4x4 case 上找到比 Strassen 更优的解 对一般创业者的工程化门槛是"评估器"——大多数业务没有这个
在 TPU 设计 RTL 优化上产出可合成的电路改进 和 OpenAI o3 的 deep-think 路线相比，谁会先成为 agent 范式存疑
HN 327 票讨论，评论中多位 ML/HPC 研究者承认结果可复现

📖 主编点评

作为做 Agent 的人你应该看的是"评估器优先"这条路：模型再强，没有自动评估循环就是聊天机器人。content-curator 里你已经有的反馈/打分循环就是一种轻量评估器，可以再往里加一层"Claude 自评 + 用户打分"的对照实验。AlphaEvolve 这种范式不能直接抄，但它给所有 Agent 项目设定了北极星：能闭环的 Agent 才会复利成长。
